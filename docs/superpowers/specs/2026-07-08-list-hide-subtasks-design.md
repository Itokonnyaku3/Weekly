# リストビュー: サブタスク非表示モード + 条件エリア横並び

日付: 2026-07-08 / 対象: v2 リストビュー

## 目的
1. リスト表示でサブタスクを非表示にできるモード（既定=非表示）。
2. 上部ビュー選択下の「条件」エリアが縦に嵩張るので横並びにして省スペース化。

## サブタスクの定義
`kind:'task'` のカードのうち、**ref 階層の祖先のどこかにタスクがある**もの＝サブタスク。
- 例: `タスクA → タスクB → タスクC` … B・C がサブタスク。
- 例: `タスクA → メモ → タスクD` … D もサブタスク（上位にタスクAがいる）。
- この「任意の祖先がタスク」定義により、下記の件数カウント（孫含む）と定義が一致する。
- カードは複数 ref を持ちうる。いずれかの ref の祖先にタスクがあればサブタスク扱い（`some`）。

## 機能A: 表示/非表示 + 件数バッジ
- チェックボックス「サブタスクを表示」を**ビュー選択バーの右端**に配置。
- 既定 OFF（非表示）。状態 `listState._showSubtasks`（undefined→false）。
- 保存ビューにも `showSubtasks` を保存/復元。既存ビューは未定義→非表示（望ましい既定なので移行不要）。
- 非表示時: サブタスク行をリストから除外。件数（`n/total 件`）は実表示行数に合わせる。
- 件数バッジ: 各タスクのタイトル右に**配下の子孫タスク総数**（>0のとき）を `（3）` で表示。
  表示/非表示に関わらず常時。編集不可の別 span（リネーム入力に混ざらない）。

## 機能B: 条件エリア横並び
- CSS `.filter-group` を `flex-direction:column` → `row; flex-wrap:wrap` に変更。
- DOM 構造は不変（期限 / 完了 / PJ・中項目・優先度 の3ブロックが横に並ぶ）。

## 実装
- `store.js`: `allRefs()` 追加（`queryBodies` の ref 版・追記のみ）。
- `list.js`: 純関数 `subtaskIndex(store)` → `{ subtaskIds:Set, descCount:Map<bodyId,number> }`
  （全ref 1パスで親子関係と本体種別を索引化し、各カードの祖先タスク判定と子孫タスク数を算出）。
  `renderList` で非表示フィルタ、`cellTitle` で `（n）` 付与（`_listCtx.descCount` 経由）。
- `app.js`: `listState` に `_showSubtasks` は状態オブジェクトに動的付与（初期は未定義=false）。
- テスト: `tests/list.subtask.test.mjs`（純ロジック）。

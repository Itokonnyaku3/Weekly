# 請求処理ビュー: タグプロパティ × リストビュー拡張（設計）

2026-07-17 / 対象: v2（v0.91.0 予定）

## 目的

「請求処理の進捗だけをまとめて見られるビュー」を作る。`#請求処理` タグの付いたタスクを
リストビューで絞り込み、見積日〜支払日の日付とステータスを Excel のような列で俯瞰・直接編集できるようにする。

## 決定事項（ヒアリング結果）

| 論点 | 決定 |
|---|---|
| プロパティの型 | 見積日/発注/請求処理/支払日=日付（空欄可）、ステータス=選択式 |
| ステータス選択肢 | 未着手→見積依頼中→発注済→請求処理中→支払済→完了（＋空欄） |
| スキーマ定義 | B案=コード内の組み込み定数。ユーザー定義UI（A案）はバックログ（docs/BACKLOG.md） |
| タグの付け方 | A案=既存どおりタイトル本文に `#請求処理` と書く（検索ビューと同一機構） |
| 運用 | 日付は予定を入れておき、確定したらステータスを変えて確定日付に置き換える |

## 設計

### 1. データモデル

- プロパティ値はタスク本体に持つ: `body.props = { quote:'2026-07-20', status:'発注済' }`。
  - キーはスキーマの `key`（フラット）。空欄はキー自体を削除し、全て空なら `props` ごと削除（JSONを太らせない）。
  - 既存コードは `props` を知らなくても壊れない（保存/GitHub同期は状態丸ごとシリアライズ）。
  - タイトルからタグを外しても値は残る（列が出なくなるだけ）。許容する。
- スキーマは新モジュール `v2/src/props.js` にコード定数 `TAG_SCHEMAS` で定義:
  - `請求処理`: quote=見積日(date), order=発注(date), bill=請求処理(date), pay=支払日(date),
    status=ステータス(select, 上記6択)
- `cardTags`（`#タグ` 抽出）は search.js から props.js へ移設し、search.js から再エクスポート。
  - 理由: search.js→list.js の import が既にあり、list.js→search.js を足すと top-level await
    動的 import の循環でデッドロックの危険があるため。

### 2. リストビューのタグ条件

- 条件グループ（`defaultGroup`）に `tags: []` を追加。
- UI: グループカード内にテキスト入力（空白区切り・`#` 不要・既存タグの datalist サジェスト）。
- 判定: グループ内 AND（指定タグをすべて含む）。OR は既存の「＋OR条件を追加」で表現。
- 旧保存ビューは `viewToGroups` の `{...defaultGroup(), ...g}` で自動補完され互換。

### 3. カスタムプロパティ列（動的列）

- 列キー: `p:請求処理:quote` 形式。
- 条件グループのどれかにスキーマ持ちタグが指定されている時だけ、列ピッカーにそのタグの
  プロパティ列が現れる。タグをUIで指定した瞬間に該当プロパティ列を state.columns へ自動追加（既定ON）。
- 表示位置: 期限の後・作成日の前。
- セル表示: 日付/ステータスをチップ表示（空欄は `—`）。
- ダブルクリックでその場編集: date→`<input type=date>`、select→`<select>`。
  change で `store.updateBody(id, { props })` → 既存の undo/保存に乗る。
  既存列（期限等）の編集方式（Enter→詳細ポップアップ）は変えない。
- state.columns に残った古い `p:` キーは activeColumns で「現在のスキーマタグ ∩ 保存列」に
  フィルタされ無害。

### 4. 列幅リサイズ

- 全列対象。各 `th` 右端にドラッグハンドル。ドラッグ中は DOM 直更新（再描画なし）、
  mouseup で `state.colWidths = { 列キー: px }` に確定＋`touch()`。
- テーブルは `table-layout: fixed` に切替。各列クラスに CSS で既定幅を持たせ、
  タイトル列は残り幅＋省略記号（ellipsis）。ツリー表示（colspan 見出し行）も fixed で崩れないことを目視検証。

### 5. ビュー保存への統合

- `saveView` に `colWidths` を追加（groups 内 `tags` は既存機構で自動保存）。`applyView` で復元。

### 6. スコープ外

- プロパティ列での並べ替え／プロパティ値での絞り込み（例: 支払日が空）
- 詳細ポップアップへのプロパティ欄追加
- スキーマのユーザー定義UI（バックログ済み）

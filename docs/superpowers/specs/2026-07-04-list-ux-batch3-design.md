## タスク一覧と確定仕様

### #1 Alt+←/→ でナビゲーション履歴を戻る/進む
- **ナビ履歴全般**（ビュー切替 / 日付ジャンプ / プロジェクトを開く / ズーム）をスタックに積み、**Alt+← で直前の状態へ、Alt+→ で進む**（ブラウザの戻る/進む相当）。
- 実装: `app.js` に履歴スタック（`navHist`/`navFuture`）を追加。ナビ状態を1オブジェクトで表現（`{ view, splitRight, projId, rootRef(zoom), date/focus }`）。`selectView`/`gotoDate`/`openProject`/`zoomToCard`/`jumpToCard` など「画面遷移」時に**現在状態を push**（連続同一はまとめる）。Alt+←/→ で pop/redo して状態を復元（`showView`＋zoom/proj/date を戻して `renderAll`）。
- 通常のカード編集・スクロールは履歴に積まない（画面遷移のみ）。上限（例: 50）。
- 検証: 実機evalで リスト→デイリー→PJ→Alt+←で戻る／Alt+→で進む、ズーム/日付ジャンプの往復。

### #2 Ctrl+/ で日付を挿入（IME中でも有効）
- カーソル位置に**今日の日付を YYMMDD 形式**（例 `260704`）で挿入。**キーは Ctrl+/**。IME変換中でも効くよう、`onKey` 冒頭の IME ガード（`isComposing`/keyCode 229）より**前**で捕捉する。
- 実装: `daily.js` `onKey` の先頭付近に Ctrl+/（`e.key === '/'` かつ ctrl）分岐。`serializeEditable`/`caretOffset` で現在テキストと位置を取り、日付文字列を挿入して `updateBody`＋`focusCard`（挿入後位置）。
- ※ 形式は YYMMDD を既定とする（要確認ポイント）。別形式希望なら差し替え容易。
- 検証: 実機evalで カーソル位置に `260704` が入る／IME on 相当でも動く（composition中の扱い）。

### #3 リストのPJ見出し: クリック=選択 / ▸トグルで折りたたみ / Enterでそのプロジェクトを開く
- **対象は PJ見出し（`groupRow`）のみ**（中項目見出しは従来どおりクリックで折りたたみ）。
- 変更: PJ見出し行の**クリックは折りたたみをやめ「選択（フォーカス）」**に。折りたたみは行頭の **▸/▾ トグルのクリック**でのみ行う。PJ見出しにフォーカス中に **Enter で `openProject(projId)`**（プロジェクトビューで開く。分割中は右ペイン）。
- 実装: `list.js` `groupRow` の `td.onclick = onToggle` を廃止し、トグル用 `span` に `onClick`（`e.stopPropagation()`＋onToggle）を付与。td クリックはフォーカスのみ。`td` の keydown で Enter → `openProject`（app から渡すコールバック `onOpenProject`）。`renderList`/`app.js` の配線に `onOpenProject` を追加。
- 検証: PJ見出しクリックで折りたたまれない・▸で折りたたむ・EnterでPJが開く。中項目見出しは従来どおり。

### #4 リスト: クリックで行全体を選択
- セル内の個別要素でなく、**行のどこをクリックしても行全体を選択**（`_sel` に1件＝その行、`row-sel` ハイライト）。
- ただし**操作系は維持**: チェックボックス（完了トグル）、↗（デイリーで開く）、タイトルのダブルクリック編集、各チップの詳細ポップアップ（PJ/中項目/優先度/期限）は従来どおり動く（これらの要素上のクリックは行選択に加えて本来の動作、または本来の動作優先）。
- 変更: タスク行 `tr` に `click` リスナ（`e.target` が操作系＝input/button/`.cell-open`/`.title-chip` の編集トリガでなければ）で行選択に。既存のセルカーソル（`data-col` フォーカス）とは別に、**行選択ハイライト**を主にする。矢印上下は #4 とは別（現状の動作を維持）。
- ※ #3 の PJ見出しクリック=選択 と整合（見出しは見出し選択、タスク行は行選択）。
- 検証: タスク行の余白/タイトルクリックで行ハイライト・チェック/↗/詳細は従来通り。

### #5 中項目なしプロジェクトのインデントを中項目ありと揃える
- 現状ツリー（grouped）表示は、**中項目が1つも無いプロジェクトだけ mid 見出しを出さず**、タスクのインデントが 18px（mid ありは 34px）でズレる。チェックボックスの位置も揃わない。
- **修正方針**: 中項目が全く無いプロジェクトでも **「（中項目なし）」の中項目見出しを必ず作り**、全プロジェクトで mid 見出し＋34px インデントに統一する（＝ユーザー要望どおり「中項目なし、という中項目を作って合わせる」）。
- 実装: `renderList` の集計プリパスを `if (t.mid) projHasMid[g] = true;` → **`projHasMid[g] = true;`**（grouped の全グループを mid あり扱い）に変更。これにより 各PJに `（中項目なし）` の `midRow` が出て、`addRow`（追加ボタン）は mid 見出し側に一本化、タスク indent は一律 34px。PJ見出し側の 18px 追加行分岐は発火しなくなる。
- 検証: 中項目なしPJでも `（中項目なし）` 見出しが出てタスクが 34px にインデント、mid ありPJとチェックボックス位置が揃う。↗ とチェックの重なりも解消することを実機確認。

### #6 割当カード（📌）に簡易フィルタを追加
- プロジェクトのノートページ下部「📌 割当カード」セクションに、**キーワード検索／完了を隠す／期限で絞る** の簡易フィルタを追加。
- 実装: `project.js` の `renderMirrorSection`（`collectMirrorRoots` で集めたルート群を描画）に、上部へフィルタUI（テキスト入力＝キーワード部分一致、チェック＝完了を隠す、簡易な期限絞り）を追加。フィルタ状態はセッション（`projState` か mirror 用 state）に保持。集約対象のルート（＝タスク本体）に対して キーワード（content 部分一致）・完了（`done`）・期限（`due`）で絞ってから描画。
- ミラーは実体編集を維持（フィルタは表示の絞り込みのみ）。
- 検証: キーワードで絞る・完了を隠すで完了ミラーが消える・期限絞りが効く。

### #8 段階的カスケード折りたたみをデイリーにも
- デイリー/プロジェクトのアウトラインでも、**Ctrl+↑ で段階的にカスケード折りたたみ**（Ctrl+↓ で展開）できるように。
  - 現状のデイリー `onKey` の Ctrl+↑↓ は「そのカードの `ref.collapsed` をトグル」するのみ。
  - 改修: フォーカス中カードで Ctrl+↑ → **未折りたたみの子があれば、まず子を畳む → 次段（親/兄弟の階層）へ** といった段階的挙動（リストの `collapseKey` の思想をアウトラインに適用）。具体挙動は実装時に現状把握のうえ、リスト相当の「深い所からたたむ／たたみ済みなら一段上」を定義。
- 実装: `daily.js` の Ctrl+↑↓ 分岐を段階化。ツリー構造（`store.childRefs`/`ref.collapsed`）に基づき対象を決定。
- 検証: 実機evalで 子あり→Ctrl+↑で子を畳む→さらにCtrl+↑で親…、Ctrl+↓で逆展開。

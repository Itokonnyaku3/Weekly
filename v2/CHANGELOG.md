# Tracker v2 — CHANGELOG

## v0.52.0 — ⋯行メニューの Tab フォーカス移動＋フォーカス維持（2026-06-30）

- **⋯メニューで項目を変更してもフォーカスが消えない**ように修正。これまでは 優先度などの値を変えると再描画でメニューが作り直されフォーカスが `<body>` に飛んでいた。再描画後に同じ項目へフォーカスを戻す（各コントロールに `data-mk` キー＋`refocusMenu`）。
- **⋯メニュー内で Tab / Shift+Tab が各項目をリング状に巡回**（メニュー外へ出ない）。
- **カード（タイトル行）にカーソルがある状態でそのカードの⋯メニューが開いていれば、Tab でメニューへフォーカス**を移す（インデントしない）。
- 実装: `daily.js` `buildCardMenu`（Tabリング＋`data-mk`＋変更後 `refocusMenu`）、`onKey` の Tab（`_openMenu===ref.id` ならメニューへ）。
- 検証: 実機evalで 優先度変更→フォーカス維持（値も反映）・メニュー内Tab巡回（優先度→期限）・カードからTabでメニューへ を確認。（リスト詳細ポップアップの Tab リングは v0.51 で対応済・動作確認済）

## v0.49.0 — 取り消し / やり直し（Ctrl+Z / Ctrl+Y）（2026-06-30）

- **Ctrl/⌘+Z で取り消し、Ctrl/⌘+Y（または Ctrl+Shift+Z）でやり直し**。全ビュー（デイリー/リスト/プロジェクト/表）の編集・追加・削除・並べ替え・属性変更に対応。
- 履歴はストアのスナップショット方式。**変更を 500ms デバウンスで coalescing** ＝連続入力や「1操作内の複数変更」（例: Enter分割＝本文切詰め＋新カード）を**1ステップ**にまとめる。最大40ステップ。
- 新しい変更を行うと redo はクリア。GitHub取得などの `replaceState` 後は履歴をリセット。
- 実装: `store.js`（`_commit`/`_scheduleCommit`/`_applyState`・`undo`/`redo`/`canUndo`/`canRedo`/`commitHistory`、`emit` がコミットをスケジュール）、`app.js`（Ctrl+Z/Y 配線＋トースト）。単体テスト `store.history`。
- 検証: 単体（編集→取消→カード作成取消→やり直し・連続変更が1ステップ・新変更でredo無効・replaceStateでリセット）＋実機evalでキー操作とDOM反映を確認。
- 既知の制限: スナップショット方式のためデータが非常に大きいと履歴のメモリが増える（上限40で抑制）。

## 📋 メール・通知の優先度判断ルール

### 優先度：低（自動アーカイブ候補）
- メルマガ・外部サービスからの一斉配信（例：ビジネス+IT、日経新聞、じゃらん等）
- 「春のキャンペーン」「会員情報変更確認」などの定型通知
- 社内の業務外連絡（歓迎会・懇親会のお知らせ等）※ただし返信要否を確認
  - **例外**: 会場・日時・場所の情報が含まれる場合は「中」に引き上げて保持する（例：名雪さん歓迎会のお知らせ → 会場情報があるため保持）

### 優先度：高（即日対応）
- 店舗からの機器トラブル・障害報告
- 上長・関係部署からの明示的な依頼
- 納期・工事・調査など日程が決まっているもの

## 🏷️ メール区分の定義

| 区分 | 説明 | 例 |
|------|------|-----|
| 依頼 | 何らかのアクションを求められている | 納期確認、承認依頼、資料提出 |
| 情報共有 | FYIで送られてくる情報 | 障害情報、会議メモの共有 |
| 確認 | 返信・反応が必要な確認事項 | 日程調整、内容の確認依頼 |
| 通知・自動配信 | システムや外部サービスからの自動送信 | ソラカメ通知、セキュリティアラート |
| 社内連絡 | 業務外含む社内からのお知らせ | 歓迎会、キャンペーン案内 |

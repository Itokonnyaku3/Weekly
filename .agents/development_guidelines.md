# Gemini 開発ガイドライン (project-weekly-tracker)

本プロジェクトにおける、AI開発アシスタントが遵守すべき設計・開発ルールです。

## 1. Single Source of Truth (SSoT) の遵守
UIの変更は、直接DOMを書き換えるのではなく、必ずステートオブジェクト（`S` 変数）を更新した後、`render()` 関数を呼び出して描画を同期させるフローを徹底してください。
データと表示の乖離はバグの温床となるため、原則としてDOMへの直接アクセスは禁止します。

## 2. 既存機能の全量スキャン
コードを修正する前に、修正対象の関数や変数が、他の箇所（特に `render`、`olRender`、`Maps` 関連の関数内）でどのように使用されているかを必ず全スキャンしてください。
一つの修正が複数の表示ロジックやイベントハンドラに影響を与える可能性があるため、依存関係を壊さないように慎重に作業してください。

## 3. 共通定数・データ構造の保護
- ストレージキー (`pwt_s`, `pwt_rp` など) を勝手に変更しないでください。
- フォーカスキーや一意キーの形式 (`pi:wk:ei`) はシステム共通で利用されているため、これを独自形式に変更しないでください。

## 4. 関数単位の提示
修正案を提示する際は、「〜行目を書き換える」といった断片的な差分ではなく、**「修正後の関数全体」**を提示してください。これにより、文脈の欠落やコピペミスによるシンタックスエラーを防止します。

## 5. 更新履歴の自動更新
修正が完了するたびに、ソースコード冒頭近くにある**「仕様・実装メモ」**コメントを最新の状態に更新してください。
何が変更され、どのような設計判断が行われたかを常にコード内に残してください。

## 6. 自動同期 ( Git Commit & Push )
個別の修正が完了した際は、[.agents/sync_policy.md](file:///c:/Users/gobou/Desktop/AntigravityWork/.agents/sync_policy.md) に基づき、Antigravity が自動的にコミットおよびプッシュを行います。
ユーザーによる手動の同期作業（バッチファイルの実行など）は原則不要です。

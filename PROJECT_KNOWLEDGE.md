# Project Weekly Tracker 開発・保守用ナレッジベース

このドキュメントは、AI（Antigravity）が本プロジェクトの修正を行う際に、過去の修正意図（デザインのこだわり、バグ修正の経緯など）を把握し、デグレ（後退）を防ぐための「記憶」として機能します。

## 🚨 共通の厳守事項（デグレ防止）

### 1. デザインと垂直整列
*   **アイコン/チェックボックス幅**: すべてのアイテムの左端をピシッと揃えるため、タグ（アイコン）およびチェックボックスのコンテナ幅は **20px** に固定すること。
*   **タグの見た目**: 枠線や背景色は、ユーザーの要望により **撤去済み**。アイコンのみを表示し、余計な装飾を加えないこと。

### 2. コマンドパレット (Ctrl+K)
*   **IMEオフ設定**: ローマ字検索をスムーズにするため、入力欄 (`#cmd-inp`) は `type="url"`, `inputmode="url"`, `style="ime-mode:disabled"` を維持すること。(`type="text"` に戻すとIMEがオンになる問題がある)
*   **ナビゲーション**: 上下キーで選択を循環し、Enterで実行。選択中の項目は `accent-soft` の背景色と左側のネイビー線で目立たせること。

### 3. グリッド（表）のフォーカス管理
*   **繰り返しアイテムの追従**: 左右キーで前後の週に移動する際、同じ繰り返し項目を指し示すために仮想インデックス `data-ei="r{rei}"` を活用。`navigate` 関数や `applyFocus` で、この文字ベースのインデックスを維持すること。
*   **パスワード保存ポップアップ防止**: GitHub PAT入力欄には `autocomplete="new-password"`, タスク入力欄には `autocomplete="off"` を設定し、ブラウザの誤検知を防ぐこと。

### 4. デイリーノート（アウトライナー）
*   **上下キー移動の安定化**: 移動（ArrowUp/Down）の「直前」に必ず `olSaveTxt` を呼び出し、編集内容をデータに反映させること。
*   **描画スキップの回避**: `olRender` の `renderKey` に必ず `_olFocusId` と `_olFocusAtStart` を含めること。これを含めないと、内容が変わっていない場合に「移動後のフォーカス移動」がスキップされてしまう。


## 🌏 将来の展望：ノード統合 (Node-Centric Architecture)

既存の「グリッドエントリ」と「ノート（ノード）」を分離させず、**すべてを「ノード」に統一する** 方向でリファクタリングを進めている。

### 基本的な考え方
1.  **すべてはノード**: 全データは日付（Daily Note）またはプロジェクト（Project Note）に紐づくノード階層として保存する。
2.  **Weeklyは「フィルタリングビュー」**: Weeklyグリッドは、「特定のプロジェクトタグ（例: `#ProjectA`）」が付いたノードを各ノートから抽出して表示するだけの「窓」となる。
3.  **属性は「タグ」**: 優先度や期限は `#prio:high`, `#due:0327` のようなタグで表現し、ノート上では通常時は **目立たない（薄い・非表示）** ようにスタイリングする。
4.  **自動作成**: Weeklyに追加したアイテムは、自動的にデイリーノート（本日または週の月曜）にノードとして作成されるようにする。

## 🔧 ノード統合 第1段階の実装状況（2026-03-27）

### 完了した実装
1.  **`setTagFilter()` 関数**: Weeklyグリッド上のタグチップをクリックすると、そのタグでフィルタリングが可能。ツールバーにインジケータが表示され、クリックで解除。
2.  **属性タグの3段階スタイリング**:
    - `.ol-tag` — 汎用タグ（opacity:0.4 → hover時 1.0）
    - `.ol-tag-attr` — 属性タグ（`#prio:`, `#due:`, `#done` 等、opacity:0.15 → hover時 0.5）
    - `.ol-tag-proj` — プロジェクトタグ（opacity:0.3 → hover時 0.8、アクセントカラー）
3.  **プロジェクトノートの器**: `S.dailyOutline['proj:' + pi]` にプロジェクト固有ノードを保存。Weekly の📄ボタンからアクセス可能。
4.  **quickAdd / savePanel のノード同期改善**:
    - 当週 → 本日のデイリーノート、他の週 → その週の月曜日のデイリーノートにノード作成
    - `projTag` プロパティでプロジェクト帰属を管理（テキストにタグを直接入れない）
    - `wkeyToDate()` がDateオブジェクトを返す問題を修正（文字列変換を追加）
5.  **projTagバッジ**: アウトライナー上で、リンクなしノードに `projTag` がある場合に控えめなプロジェクトタグバッジを表示。

### 次のステップ（第2段階）
- Weeklyの描画ソースを「entries」から「ノードの集計結果」に切り替え（二重書き込み期間終了後）

- 属性ノードの自動生成（`#task` タグ付与時に `#prio:`, `#due:` を自動付加）

## 🔧 機能追加・バグ修正ログ（2026-03-31）

### 1. ノード別日移動コマンド（Ctrl+.）
- スラッシュメニュー（旧: テキスト入力の `/`）を **Ctrl+.** ショートカットに変更
- スラッシュメニューに「別の日に移動」サブメニューを追加
  - 明日・明後日・来週・昨日のクイックボタン ＋ 日付入力フォーム
- `olMoveNodeToDate(nodeId, fromDate, toDate)`: サブツリーごと移動し、対象週が異なる場合は `S.wOff` を自動調整
- **注意**: スラッシュメニュー表示位置はビューポート境界を考慮して補正済み（画面下・空contenteditable両方対応）

### 2. ノートアウトライン Alt+Shift+↑↓ の移動ロジック修正
- 移動先を「同インデントの前後ノード（ピア）」のみに限定
- 異なるインデントのノードを引きずらないよう、上方向は `indent < myIndent` で停止、下方向も同様

### 3. サブタスク件数バッジ（ノートアウトライン）
- `parentId` リンクを持つ子ノードがある場合、親ノードの行末に **(完了数/総数)** バッジをインライン表示
  - 完了なし: `(N)` / 完了あり: `(done/N)`
- **Ctrl+Enter** または バッジクリックで、そのノードの下に `↳` 参照行を展開表示
- フォーカスが離れると自動で折りたたむ（`onfocus` で `_olRefRowsExpanded = false` リセット）
- 状態変数: `let _olRefRowsExpanded = false`（boolean、フォーカス中ノード1件のみ）
- **デグレ注意**: `renderKey` に `_olRefRowsExpanded` を含めること

### 4. Enter キー分割時の紐づけ移動（ノートアウトライン）
- ノードの**文中**でEnterを押したとき、カーソル後半のテキストが新ノードに分割される際に以下も新ノードに移動:
  - `tags`（ノートタグ配列）
  - `projTag`（プロジェクト紐づけ）
  - `parentId` リンクを持つ他日付子ノード（全 `S.dailyOutline` をスキャンして `parentId` を付け替え）
- **文末**でのEnterは従来通り（新ノードは空、紐づけは元ノードに残る）

### 5. グリッド Alt+Shift+↑↓ 移動後のフォーカス消失修正
- **原因①（別日付移動）**: `focusKey` が旧列（`wk`）を指したまま `applyFocus` を呼んでいたため、移動先でアイテムを見つけられずフォーカスが消えていた
- **原因②（同日付移動）**: `render()` の二重 rAF による `applyFocusKey` 復元が不安定
- **修正**: 両ケースとも `render()` 直後に `requestAnimationFrame(() => applyFocus(pi, targetWk, ei))` を明示呼び出し
  - 別日付の場合: `targetItem.date` から `wkey(new Date(...))` で正しい週キーを計算

## 🛠 実装の重要メモ

### テーマ設定 (Dark Mode / Theme)
*   `:root` に適用される `light-theme` / `dark-theme` クラスにより、CSS変数をオーバーライドしている。OS設定（Auto）を優先しつつ、手動設定がある場合は `pwt_theme` (localStorage) とクラス付与で強制する。

### アウトライナーのリンク機能
*   ノードをプロジェクトやタスクに紐づける機能は、`linkedEntryRef` に保存される。移動や描画の際、このリンクが壊れないよう `olSaveTxt` での解釈に注意すること。
